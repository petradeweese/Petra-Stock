What the Scanner Does

The scanner evaluates a universe of symbols to surface option setups.

  • Fetches price and volatility data for the selected symbols.
  • Calculates indicators and pattern statistics for each window.
  • Ranks results by expected return, hit rate and stability.

Parameters Explained

  • Scan Type – Universe to scan. Valid: Top150, Top250, SP100 or Single. Default: Top150. Larger universes take longer.
  • Ticker – Symbol used when Scan Type is Single. Must be a tradable ticker. Empty by default.
  • Interval – Bar size for analysis (5m, 15m, 30m, 1h). Shorter bars react faster but add noise.
  • Direction – BOTH, UP or DOWN. Determines long/short bias. BOTH checks both sides but doubles work.
  • Target % / Stop % – Profit target and stop loss in percent. Defaults 1.0/0.5. Wider targets reduce hits but increase reward.
  • Within – Time allowed to hit the target. Specify value and unit (Hours/Days). Default 4 Hours. Tight windows filter out slower setups.
  • Lookback (years) – History depth used to estimate stats. Default 2 years. More history may smooth results but increases compute.
  • Max TT Bars – Maximum bars a trade can take to reach target. Default 12. Higher values allow longer trades.
  • Min Support – Minimum historical occurrences for a pattern. Default 20. Lower values include scarce data.
  • Delta (assumed) – Estimated option delta for ROI math. Default 0.40. Adjust for moneyness sensitivity.
  • Theta per day % – Expected daily decay of option price. Default 0.20. Higher values penalize time.
  • ATRz gate – Minimum ATR z-score. Default 0.10. Filters out low-volatility conditions.
  • Slope gate % – Minimum trend slope percent. Default 0.02. Larger slopes require stronger trends.
  • Use Regime / Regime Trend Only – Apply market regime filters. Default Off. Trend only restricts to regime trend direction.
  • VIX z max – Maximum VIX z-score allowed. Default 3.0. Lower values avoid extreme volatility.
  • Slippage (bps) – Estimated execution cost in basis points. Default 7. Higher slippage reduces ROI.
  • Vega scale – Scaling factor for implied volatility impact. Default 0.03. Adjust for option sensitivity.
  • Scan min Hit % – Minimum historical hit rate. Default 50%. Raising this tightens quality.
  • Scan max DD % – Maximum drawdown allowed. Default 50%. Lower limits cut riskier patterns.
  • Email results – Send summary email after a run. Default No. Requires SMTP settings.

How Results Are Computed

  1. Retrieve price and volatility data for the chosen universe.
  2. Preprocess and align data into analysis windows.
  3. Compute indicators and detect pattern matches.
  4. Score and filter candidates by rules and thresholds.
  5. Sort the remaining rows by ROI, hit rate and stability.
